Git

Controle de versão é um problema antigo: imagine diversos programadores trabalhando no mesmo projeto, cada um na sua funcionalidade, no seu tempo e separados, ou você mesmo trabalhando em seu projeto e pretende trabalhar em funcionalidades distintas, depois precisa juntar isso tudo num código final que traz todo trabalho que todos fizeram.

Ou então você precisa desfazer um trabalho ou voltar para um momento em que um bug ainda havia sindo inserido no programa.

O software que cuida disso é o sistema de controle de versões.

CVS

Dentre os mais usados, o primeiro sistema de controle de versões foi o CVS, Concurrent Versions System, criado em 1986 e foi a única opção razoável para versionamento concorrente.

Subversion

Até que em 2001 o projeto Apache liberou o Subversion, ou SVN, criado em 2000.

A partir de então, diversos projetos foram criados para concorrer com o Subversion e o CVS.

Git

Em 2005, Linus Torvalds, criador do Linux, desenvolveu o Git, o sistema de controle de versões mais eficiente que há, inicialmente como apenas uma coleção de scripts, mas evoluindo para uma aplicação CLI completa que praticamente tomou o lugar de todos os concorrentes.

Git é o motor de diversos repositórios online, como:

  • GitHub
  • GitLab
  • Bitbucket
  • SourceForge
  • Assembla
  • CodeCommit
  • Azure DevOps

Descentralizado

A grande diferença do Git pra seus antecessores é que ele é descentralizado, ou seja, não existe um repositórios central ao qual todos os outros devem reportar – todos os repositórios são iguais.

Ferramentas de apoio

Quando você instala o Git, costuma ser instalada junto uma ferramenta gráfica chamada gitk, baseada na libtk, uma biblioteca para construção de artefatos gráficos para X11, como janelas, etiquetas e botões, mas a maioria dos programadores não gosta do gitk.

É comum que instalem em seu lugar outra ferramenta gráfica chamada GitKraken.

Pessoalmente eu recomendo duas ferramentas TUI que considero mais simples e úteis, lazygit e gitui.

Alguns comandos

Esta é uma lista muito superficial de recursos do Git:

  • git init: inicializa um novo projeto.
  • git clone: clona um projeto de outro repositório, criando-o localmente.
  • git add: adiciona um ou mais arquivos ao projeto.
  • git commit: salva as alterações no repositório, tornando possível compartilhá-las com outros repositórios.
  • git status: exibe o estado dos arquivos no projeto.
  • git push: envia as alterações do projeto para outro repositório.
  • git pull: busca as alterações do projeto de outro repositório.
  • git log: exibe a lista de commits.
  • git branch: gerencia branchs (troncos de trabalho paralelo).
  • git checkout: alterna entre branchs e commits.
  • git rebase: reorganiza os commits.
  • git cherry-pick: move commits entre branches.
  • git blame: busca por commit(s) que inseriu um bug.

lazygit e gitui possuem subcommandos equivalentes.
