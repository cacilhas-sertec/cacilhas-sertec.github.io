Java

Java é uma linguagem de programação criada em 1993 por James Gosling a partir da linguagem Oak de 1991, também criação de Gosling.

O primeiro release de Java foi lançado em 1996 pela Sun Microsystems.

As versões LTS são Java 8, Java 11 e Java 17, então é seguro usar qualquer dessas versões em suas aplicações de produção.

Princípios

Java foi criada seguindo cinco princípios básicos:

  1. Ser orientada a objetos;
  2. Ser robusta e segura;
  3. Ser portátil, agnóstica à arquitetura;
  4. Garantir alta performance;
  5. Ser interpretada, dinâmica e ter suporte a threads.

Para ganrantir que Java seja portátil (agnóstica à arquitetura), ela não roda diretamente no sistema operacional de base, mas sim numa máquina virtual, chamada JVM. Isso é ser um linguagem interpretada.

Atualmente não há apenas uma JVM, mas algumas, sendo as mais importantes:

  • a JVM da Oracle;
  • OpenJDK;
  • GraalVM.

“Olá Mundo!!”

Sem pensar em arquitetura de aplicação e estrutura de pacotes, podemos criar um código “Olá Mundo!!” bastante simples, apenas com um arquivo. Vamos criar um arquivo chamado HelloWorld.java – é importante que o nome do arquivo antes da extensão .java seja extamente o nome da classe pública que criaremos dentro dele: 

public class HelloWorld {
    public static void main(String[] args)  {
        System.out.println("Olá Mundo!!");
    }
}

public class declara uma classe pública, que, como dito antes, precisa ter o mesmo nome do arquivo. O método main, que deve ser público e estático (ou seja, executado pela classe, não por instâncias dela), é o entrypoint da aplicação, e deve receber os parâmetros de linha de comando, que são uma lista de strings (String[]).

O método println de System.out exibe um texto na saída padrão

Todos os statements devem ser encerrados por ponto-e-vírgula (;).

Para compilar o programa para bytecode, o formato executável pela JVM, usamos o comando javac: 

javac HelloWorld.java

Isso criará o arquivo HelloWorld.class, que é o bytecode.

E para rodá-lo com a JVM usamos:

java HelloWorld

Caso tivéssemos um pacote java (.jar) executável, o comando requer um parâmetro extra:

java -jar $ApplicationName.jar

Baixe o código.

Bibliotecas que facilitam a vida

Apesar da fama de uma linguagem difícil, há diversas bibliotecas e frameworks que deixam tudo mais fácil, como por exemplo:

  • Lombok: traz diversas features ausentes em Java e ainda elimina a necessidade de boilerplates e construções repetitivas. Você encontra uma lista dessas funcionalidades aqui.
  • Hibernate: um ORM para lidar com bancos de dados.
  • Apache Log4J: uma biblioteca para simplificar a gerência de mensagens de log.
  • Spring: um completíssimo framework para programação orientada a eventos com suporte direto ao desenvolvimento de aplicações em rede. Faz parte do Spring o famoso Spring Boot, um framework para criação de aplicações standalone.

Outras linguagens JVM

No entanto o ecossistema Java não se limita à linguagem de programação Java: a JVM suporta diversas linguagens, que podem ser usadas isoladamente ou (algumas, não todas) integradas umas às outras.

Alguns exemplos de linguagens que rodam na JVM:

  • Closure: uma implementação de LISP Para JVM. HelloWorld.clj
  • Groovy: uma espécie de Java simplificado. HelloWorld.groovy
  • Ioke: uma linguagem baseada em Io, mas com significantes melhorias. HelloWorld.ik
  • JRuby: apenas um conector para permitir que aplicações Ruby on Rails possam rodar na JVM.
  • Jython: Python para jVM.
  • Kotlin: linguagem com foco na criação de aplicações para celulares Android. HelloWorld.kt
  • Scala: linguagem funcional impura com suporte a álgebra de tipos e tipos dependentes, em minha humilde opinião, a mais poderosa linguagem para JVM. HelloWorld.scala
  • Xtend: linguagem do projeto Eclipse construída sobre a ferramenta Xtext. Na verdade um Java menos verborrágico. HelloWorld.xtend
