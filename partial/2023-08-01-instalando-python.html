Instalando Python

Para nossos exemplos e exercícios usaremos Python como linguagem de programação na maioria das vezes, portanto é importante que tenhamos uma máquina virtual de Python instalada.

Distribuições GNU/Linux baseadas em apt

Vamos começar instalando a CPython1 no Debian GNU/Linux, que usado como gerenciador de pacotes o apt. Outras distros, como o Ubuntu e seus variantes, também utilizam o mesmo gerenciador, portanto a instalação é idêntica.

Para buscar por um pacote usamos o subcomando search. Para Buscar por Python, podemos executar o seguinte comando:

apt search python3

Isso retornará uma lista onde poderemos encontrar algumas versões de Python, com provavelmente algo assim:

python3.11-full/testing 3.11.4-1 amd64
  Python Interpreter with complete class library (version 3.11)

Na primeira linha vemos o nome do pacote (python3.11-full), a versão do sistema (testing), a versão do pacote (3.11.4-1) e a arquitetura (amd64).

Para instalar esse pacote usamos o subcomando install, no entanto precisamos prefixar o comando com sudo, para executá-lo como usuário administrador (root):

sudo apt install -y python3.11-full

Provavelmente pedirá a senha do seu usuário e, se seu usuário for sudoer (aquele que pode requerer permissões de adminstrador), então o CPython será instalado. Você pode confirmar isso após o final da instalação executando o comando python.

Para sair do CPython basta pressionar Control+D (C-d no jargão) ou digitar exit() e pressionar Enter.

Para nossos testes, usaremos um REPL2 alternativo com uma série de funções extras, como auto-complete, chamado Ptpython. É fácil instalá-lo no Debian:

sudo apt install -y ptpython

Para gerência de versionamento dos códigos, vamos usar o Git, também facilmente instalável:

sudo apt install -y git

Você também pode instalar todos esses pacotes em um só comando:

sudo apt install -y python3.11-full ptpython git

Distribuições GNU/Linux baseadas em pacman

Outra família de distribuições GNU/Linux importante é a do Arch Linux, que utiliza o gerenciador de pacotes pacman.

Apesar de não ser tão simples quanto usar apt, também não chega a ser ciência de foguete.

Para instalar o CPython, vamos ter de instalar três pacotes, mais o Git:

sudo pacman -Syu python python-pip python-pipx git

Os parâmetros -Syu são os parâmetros de instalação, o pacote python é o CPython com Python 3.11 (ou mais recente, caso já tenha sido atualizado), python-pip é o gerenciador de pacotes de Python e python-pipx é o gerenciador de pacotes extras.

Agora vamos instalar o Ptpython, que pode ser apenas para o usuário, não de sistema. Para tanto usaremos o pipx:

pipx install ptpython

Windows

Para instalar Python, Ptpython e Git no Windows vamos usar um gerenciador de pacotes chamado Scoop.

Para instalar o Scoop, abra o PowerShell e executa os comandos:

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
irm get.scoop.sh | iex

Instalado o Scoop, buscamos por Python com:

scoop search python

Vai aparecer Python 3.11 na lista, que provavelmente será o pacote python, então, para instalar:

scoop install python

No Windows não precisamos do pipx para instalar o Ptpython, podemos instalá-lo com o próprio pip:

pip install ptpython

E por último o Git:

scoop install git

Bônus: Distribuições GNU/Linux baseadas em RPM

Outra família importante de distros GNU/Linux é RPM, da qual fazem parte RHEL, CentOS, Fedora Core e SuSE OpenLinux.

Para instalar Python e Git nesses sistemas usamos o yum: 

yum install python3 git

Até o momento não há instruções de instalação do Ptpython em distros RPM, conforme explicado no PR #386 do Ptpython.

1CPython: máquina virtual de Python escrita em C. Outras possíveis formas de rodar Python são PyPy (interpretador escrito em RPython), Jython (compilador Python para a máquina virtual Java), GraalPy (interpretador Python escrito na GraalVM) e RustPython (compilador JIT de Python escrito em Rust).

2REPL: read-eval-print loop, laço de leitura, execução e exibição, também chamado prompt.
